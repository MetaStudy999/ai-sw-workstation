# 파일 권한 입문 가이드

이 문서는 터미널을 처음 배우는 사람이 파일 권한을 어렵지 않게 이해할 수 있도록 정리한 입문용 안내서입니다.

먼저 한 가지를 짚고 가면, 터미널에 `permision`이라는 단일 명령어가 따로 있는 것은 아닙니다. 보통 "권한을 확인하고 바꾸는 작업"을 할 때 `ls -l`, `chmod`, `chown`, `chgrp` 같은 명령어를 함께 사용합니다.

이 문서의 목표는 다음과 같습니다.

- 파일 권한이 왜 필요한지 이해한다.
- 권한을 읽는 방법을 먼저 익힌다.
- 가장 자주 쓰는 권한 명령어를 예제와 함께 익힌다.
- 초보자가 어떤 순서로 응용하면 되는지 감을 잡는다.

실습은 macOS/Linux의 `zsh` 또는 `bash` 기준으로 설명합니다.
예제는 이 저장소 루트 디렉터리에서 실행한다고 가정합니다.

---

## 1. 파일 권한이란?

파일 권한은 "누가 이 파일을 읽고, 수정하고, 실행할 수 있는지"를 정하는 규칙입니다.

예를 들어 권한이 필요한 이유는 아래와 같습니다.

- 중요한 파일을 아무나 수정하지 못하게 막기 위해
- 실행 파일만 실행되게 하기 위해
- 협업할 때 소유자와 사용자 범위를 구분하기 위해

쉽게 말하면 파일 권한은 파일에 걸린 출입 규칙이라고 생각하면 이해가 쉽습니다.

---

## 2. 먼저 알아두면 쉬운 핵심 개념

### 2-1. 권한을 보는 대상 3가지

권한은 보통 아래 3개 기준으로 나누어 봅니다.

- 소유자(`user`): 그 파일을 만든 사람 또는 현재 소유한 사용자
- 그룹(`group`): 같은 그룹에 속한 사용자들
- 기타(`others`): 그 외 나머지 사용자들

예를 들어 어떤 파일의 권한이 `644`라면:

- 소유자: 읽기, 쓰기 가능
- 그룹: 읽기만 가능
- 기타: 읽기만 가능

### 2-2. 권한 문자 3가지

- `r`: read, 읽기
- `w`: write, 쓰기
- `x`: execute, 실행

### 2-3. 숫자 권한 읽는 법

숫자 권한은 아래 규칙으로 계산합니다.

- `r = 4`
- `w = 2`
- `x = 1`

합쳐서 읽으면 됩니다.

- `7 = 4 + 2 + 1 = rwx`
- `6 = 4 + 2 = rw-`
- `5 = 4 + 1 = r-x`
- `4 = 4 = r--`

그래서:

- `644 = rw- r-- r--`
- `755 = rwx r-x r-x`

## 3. 입문자가 가장 먼저 익힐 권한 명령어

| 목적 | 명령어 | 의미 |
| --- | --- | --- |
| 현재 사용자 확인 | `whoami` | 지금 어떤 사용자로 작업 중인지 확인 |
| 권한 보기 | `ls -l 파일명` | 파일의 권한, 소유자, 그룹 확인 |
| 폴더 권한 보기 | `ls -ld 폴더명` | 폴더 자체 권한 확인 |
| 권한 변경 | `chmod 권한 파일명` | 읽기/쓰기/실행 권한 변경 |
| 소유자 변경 | `chown 사용자 파일명` | 파일 소유자 변경 |
| 그룹 변경 | `chgrp 그룹 파일명` | 파일 그룹 변경 |
| 관리자 권한 사용 | `sudo 명령어` | 관리자 권한으로 명령 실행 |

초보자는 이 순서로 익히면 좋습니다.

`whoami -> ls -l -> chmod -> chown/chgrp -> sudo`

이 순서를 추천하는 이유는 "먼저 확인하고, 그다음 최소한만 바꾸는 습관"이 중요하기 때문입니다.

## 4. 가장 쉬운 실습 순서

### 4-1. 내가 누구로 작업 중인지 확인하기

```bash
whoami
```

이 명령어는 지금 어떤 사용자 계정으로 작업 중인지 보여 줍니다.

권한 문제는 "누가 작업하고 있는가"와 연결되기 때문에, 초보자는 권한 작업 전에 `whoami`를 먼저 보는 습관이 좋습니다.

### 4-2. 연습용 파일과 폴더 만들기

```bash
touch permission_file.txt
mkdir permission_dir
```

이 저장소에는 이미 [`permission_file.txt`](/Users/metastudy9997479/codyssey/ai-sw-workstation/permission_file.txt)가 있으므로, 파일은 그대로 사용해도 됩니다.
폴더는 없으면 새로 만들어서 실습하면 됩니다.

### 4-3. 권한 확인하기

파일 권한 확인:

```bash
ls -l permission_file.txt
```

폴더 권한 확인:

```bash
ls -ld permission_dir
```

예를 들어 결과가 아래처럼 보일 수 있습니다.

```bash
-rw-r--r--  1 user  staff  0 Apr 24 12:00 permission_file.txt
drwxr-xr-x  2 user  staff  64 Apr 24 12:00 permission_dir
```

읽는 방법:

- 첫 글자가 `-`이면 파일
- 첫 글자가 `d`이면 디렉터리
- 그 뒤의 `rw-r--r--` 같은 부분이 권한
- 그다음 `user`는 소유자
- `staff`는 그룹

## 5. `chmod`를 가장 먼저 익히기

초보자가 권한 작업에서 가장 먼저 익혀야 할 명령어는 `chmod`입니다.

### 5-1. 숫자로 권한 바꾸기

파일을 `644`로 바꾸기:

```bash
chmod 644 permission_file.txt
```

의미:

- 소유자: 읽기, 쓰기 가능
- 그룹: 읽기만 가능
- 기타: 읽기만 가능

폴더를 `755`로 바꾸기:

```bash
chmod 755 permission_dir
```

의미:

- 소유자: 읽기, 쓰기, 실행 가능
- 그룹: 읽기, 실행 가능
- 기타: 읽기, 실행 가능

초보자 기준으로 가장 많이 보는 조합은 아래 3개입니다.

- `644`: 일반 파일에 자주 사용
- `755`: 실행 가능해야 하는 폴더나 스크립트에 자주 사용
- `600`: 나만 읽고 쓰게 제한하고 싶을 때 사용

### 5-2. 문자 방식으로 권한 바꾸기

숫자 방식이 익숙해지면 문자 방식도 같이 익히면 좋습니다.

소유자에게 실행 권한 추가:

```bash
chmod u+x permission_file.txt
```

그룹의 쓰기 권한 제거:

```bash
chmod g-w permission_file.txt
```

기타 사용자에게 읽기 권한 추가:

```bash
chmod o+r permission_file.txt
```

여기서:

- `u`: user, 소유자
- `g`: group, 그룹
- `o`: others, 기타
- `a`: all, 모두

그리고:

- `+`: 권한 추가
- `-`: 권한 제거
- `=`: 지정한 권한으로 설정

예:

```bash
chmod a=r permission_file.txt
```

뜻:

모든 사용자에게 읽기만 허용합니다.

### 5-3. 바꾼 뒤 다시 확인하기

권한을 바꾼 뒤에는 꼭 다시 확인해야 합니다.

```bash
ls -l permission_file.txt
ls -ld permission_dir
```

초보자에게 가장 중요한 습관은 아래 한 줄입니다.

**변경 전 확인 -> 변경 -> 변경 후 재확인**

## 6. 파일과 폴더 권한이 다른 이유

초보자가 많이 헷갈리는 부분입니다.

파일에서:

- `r`: 내용 읽기
- `w`: 내용 수정
- `x`: 파일 실행

폴더에서:

- `r`: 폴더 안 목록 읽기
- `w`: 내부 파일 생성/삭제/이름 변경
- `x`: 폴더 안으로 들어가서 접근

그래서 폴더는 `x` 권한이 특히 중요합니다.
폴더에 `r`만 있고 `x`가 없으면 목록은 보여도 실제 접근이 제한될 수 있습니다.

---

## 7. `chown`과 `chgrp`는 언제 쓰나?

`chmod`는 권한 자체를 바꾸는 명령어이고, `chown`과 `chgrp`는 소유자를 바꾸는 명령어입니다.

### 7-1. `chown`

```bash
sudo chown 사용자명 permission_file.txt
```

의미:

파일의 소유자를 다른 사용자로 변경합니다.

### 7-2. `chgrp`

```bash
chgrp staff permission_file.txt
```

의미:

파일의 그룹을 변경합니다.

초보자 관점에서 기억할 점:

- 권한 문제의 원인이 "읽기/쓰기/실행 가능 여부"라면 먼저 `chmod`
- 권한 문제의 원인이 "내 파일이 아니어서 수정이 안 됨"이라면 `chown`
- 팀이나 그룹 기준 접근 문제라면 `chgrp`

## 8. 초보자용 응용 순서

실제로 권한 문제가 생겼을 때는 아래 순서로 접근하면 가장 안전합니다.

### 1단계. 현재 사용자 확인

```bash
whoami
```

### 2단계. 파일인지 폴더인지 확인

```bash
ls -l 대상파일
ls -ld 대상폴더
```

### 3단계. 현재 권한 읽기

`rw-r--r--` 또는 `drwxr-xr-x` 같은 부분을 먼저 읽습니다.

### 4단계. 필요한 권한만 최소로 변경

```bash
chmod 644 대상파일
chmod 755 대상폴더
```

### 5단계. 다시 확인

```bash
ls -l 대상파일
ls -ld 대상폴더
```

### 6단계. 그래도 안 되면 소유자 확인

```bash
ls -l 대상파일
```

소유자가 내가 아니라면 `chown`이 필요한 상황일 수 있습니다.

### 7단계. 마지막에만 `sudo` 고려

관리자 권한은 강력하므로 초보자는 습관적으로 붙이지 않는 것이 좋습니다.

## 9. 입문자가 자주 쓰는 예제 모음

### 예제 1. 일반 파일 권한을 기본 형태로 맞추기

```bash
chmod 644 permission_file.txt
ls -l permission_file.txt
```

### 예제 2. 폴더를 접근 가능하게 만들기

```bash
chmod 755 permission_dir
ls -ld permission_dir
```

### 예제 3. 스크립트에 실행 권한 주기

```bash
chmod u+x run.sh
ls -l run.sh
```

### 예제 4. 나만 볼 수 있는 파일로 만들기

```bash
chmod 600 secret.txt
ls -l secret.txt
```

### 예제 5. 소유자 변경하기

```bash
sudo chown myuser permission_file.txt
ls -l permission_file.txt
```

## 10. 초보자가 특히 조심할 점

- `chmod 777`을 습관처럼 쓰지 않는 것이 좋습니다.
- `sudo`를 이유 없이 붙이지 않는 것이 좋습니다.
- 파일과 폴더 권한은 같은 의미로 보면 안 됩니다.
- 변경하기 전과 후에 `ls -l` 또는 `ls -ld`로 꼭 확인하는 습관이 중요합니다.
- 한 번에 크게 바꾸기보다 필요한 만큼만 바꾸는 것이 안전합니다.

특히 `777`은 "모두에게 모든 권한"을 주는 뜻이라 편해 보여도 보안상 좋지 않은 경우가 많습니다.

## 11. 입문자용 학습 순서 한눈에 보기

처음에는 아래 순서대로 익히면 됩니다.

1. `whoami`로 현재 사용자 확인하기
2. `ls -l`, `ls -ld`로 권한 읽는 법 익히기
3. `chmod 644`, `chmod 755` 같은 숫자 방식 먼저 익히기
4. `chmod u+x`, `chmod g-w` 같은 문자 방식 익히기
5. `chown`, `chgrp`로 소유자와 그룹 개념 익히기
6. 마지막으로 `sudo`가 왜 필요한지 이해하기

초보자 기준 핵심 한 줄 요약은 이것입니다.

**권한 문제는 먼저 보고(`ls -l`), 필요한 만큼만 바꾸고(`chmod`), 다시 확인한다.**
